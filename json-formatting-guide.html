JSON Formatting: A Developer's Complete Guide

By ToolHub Editorial · June 27, 2026 · 6 min read

JSON is everywhere — APIs, configs, logs. Learn the rules, common pitfalls, and how to format, validate and minify JSON safely.

Why JSON is everywhere

JSON (JavaScript Object Notation) is the lingua franca of the modern web. APIs return it, configs use it, databases store it, logs emit it. Despite its simplicity, syntax mistakes constantly trip up developers — especially when JSON is hand-edited.

The complete syntax rules

  • Six value types: string, number, boolean, null, array, object. That's it.
  • Strings must use double quotes, never single.
  • Object keys must be quoted strings.
  • No trailing commas: {"a":1,} is invalid.
  • No comments. Anything that looks like // or /* */ is invalid.
  • No functions, no undefined, no dates as native values — only the six types above.

Common mistakes and how to spot them

  1. Trailing comma. "Unexpected token } in JSON at position X" — check the character before the closing brace.
  2. Single quotes. "Unexpected token ' in JSON" — Find/Replace ' with ".
  3. Unquoted keys. Common when copy-pasting from JavaScript object literals.
  4. JavaScript values. undefined isn't JSON; use null or omit the key.
  5. Mismatched brackets. A formatter shows the structure visually so misplaced brackets jump out.

Format vs minify

Format (pretty-print) adds indentation and newlines. Use during development, code review, debugging.

Minify strips every space and newline. Use in production — APIs, HTML embedding, anywhere bytes matter. A typical config can shrink 40–60% just from minification.

JSON Schema for validation

When JSON is exchanged between systems, a contract is essential. JSON Schema lets you define the expected shape — required fields, types, ranges, enums — and reject malformed payloads at the boundary. Worth adopting for any production API.
