How to use the JSON formatter

  1. Paste your JSON into the input box.
  2. Click Format to pretty-print with 2-space indentation, or Minify to strip all whitespace.
  3. If your JSON is invalid, the status line shows exactly where the parser failed.

Common JSON mistakes

  • Trailing commas — {"a":1,} is invalid
  • Single quotes — JSON requires double quotes around keys and string values
  • Unquoted keys — {name:"x"} is invalid; must be {"name":"x"}
  • JavaScript comments — // and /* */ aren't JSON
  • Undefined or function values — JSON only supports string, number, boolean, null, array and object

Format vs minify — when to use each

Use format for reading and debugging — code reviews, comparing structures, fixing bugs. Use minify for transport — APIs, embedding in HTML, anywhere file size matters.
