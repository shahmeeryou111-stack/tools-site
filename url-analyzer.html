🔗 Free URL Analyzer

How to use the URL analyzer

  1. Paste the URL you want to inspect.
  2. Click Analyze URL.
  3. Read each component — protocol, host, port, path, query parameters and hash.

The parts of a URL

  • Protocol — https: or http:
  • Host — the domain (e.g. example.com)
  • Port — usually omitted; defaults to 443 for HTTPS, 80 for HTTP
  • Path — the resource on the server (e.g. /products/12)
  • Query parameters — key/value pairs after ?
  • Hash — fragment after #, used for in-page anchors and SPA routes

Why URL analysis is useful

  • Debug tracking parameters (utm_source, gclid, fbclid)
  • Verify redirect destinations
  • Spot phishing — confirm the real host before clicking
  • Build deep links for mobile apps
